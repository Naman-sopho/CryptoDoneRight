Introduction to modes of Operations in Cryptography




Encryption algorithms are in basic definition a set of rules that allow for generation of confidentiality of data. But encryption algorithms have evolved in the way they encrypt data. The various different modes of operation work in different ways and operate with input data (to be encrypted) in different ways. Here is an overview of how encryption algorithms are organized based on their functionality.


For the case of this section of the website, we will be discussing the popular examples from confidentiality and authenticated encryption modes.

Confidentiality (Encryption Only):

As the name suggests, encryption algorithms operating in this mode can only provide encryption capabilities. There are reasons why this can be considered to be a sufficient feature but it is recommended that such a decision be taken after understanding all the possible options carefully.

Block Ciphers Modes of Operation

The modes of operation of block ciphers are configuration methods that allow those ciphers to work with large data streams, without the risk of compromising the provided security.

Modes that require padding: An element's padding is the space between its content and its border. In encryption this would mean filler bits to make up remaining space in a packet field. Padding can generally be dangerous because it opens up the possibility of padding oracle attacks (learn more about these ATTQCHAS). The easiest defense is to authenticate every message before decryption (using authenticated encryption modes of operation).

ECB:
Electronic Code Book (ECB) is a mode of operation for a block cipher, with the characteristic that each possible block of plaintext has a defined corresponding ciphertext value and vice versa. In other words, the same plaintext value will always result in the same ciphertext value. Electronic Code Book is used when a volume of plaintext is separated into several blocks of data, each of which is then encrypted independently of other blocks. Learn more (a possible video link and more resources).

Security:
  • The security properties of ECB are weak.
  • It's a legacy mode of operation.
  • Should be avoided or should be used with great caution.

CBC:
Cipher block chaining (CBC) is a mode of operation for a block cipher (one in which a sequence of bits are encrypted as a single unit or block with a cipher key applied to the entire block). Cipher block chaining uses what is known as an initialization vector (IV) of a certain length. One of its key characteristics is that it uses a chaining mechanism that causes the decryption of a block of ciphertext to depend on all the preceding ciphertext blocks. As a result, the entire validity of all preceding blocks is contained in the immediately previous ciphertext block. A single bit error in a ciphertext block affects the decryption of all subsequent blocks. Rearrangement of the order of the ciphertext blocks causes decryption to become corrupted. Basically, in cipher block chaining, each plaintext block is XORed (see XOR) with the immediately previous ciphertext block, and then encrypted.

Security:
  • Care should be taken to implement the IV correctly. Cannot be the same ever and should not be a nonce.
  • Is frequently misused.
  • No important advantages over CTR mode.


Stream Cipher Modes

These modes can generate random stream of data (bits) that may or may not depend on the plaintext. In a stream cipher, each plaintext digit is encrypted one at a time with the corresponding digit of the keystream, to give a digit of the ciphertext stream. Note that these modes operate on blocks.

CTR mode:
An IV based encryption scheme. This mode of operation is really fast as it performs a lot of operations in parallel. CTR mode acts as an important block for authenticated encryption schemes. Overall, usually the best and modern way to achieve privacy-only encryption.

Authenticated Encryption:
The idea behind authenticated encryption is to combine encryption and authentication by performing encryption on a plain text and then creating a token that equals the message digest of the cipher text. This approach has been proven to be highly secure but care must be taken to make sure that it is configured correctly. From a security standpoint, authenticated encryption has proven to be secure against padding oracle attacks.

GCM:
GCM mode provides both privacy (encryption) and integrity. To provide encryption, GCM maintains a counter; for each block of data, it sends the current value of the counter through the block cipher. Then, it takes the output of the block cipher, and exclusive or's that with the plaintext to form the ciphertext.

Security:
  • Widely standardized and used.
  • Very efficient while providing good provable security (although can get complex to implement).
