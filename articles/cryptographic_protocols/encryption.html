General Information:

Different modes of Encryption:
The two different flavors of encryption are block ciphers and stream ciphers. On a high level: A typical stream cipher encrypts plaintext one byte at a time and a block cipher operates on one block at a time. The block may be of size one byte or more or less. That means we can also encrypt a block of one byte by help of a stream cipher as a stream. For more information, visit here: https://crypto.stackexchange.com/questions/5333/difference-between-stream-cipher-and-block-cipher

Examples of Stream Cipher: RC4
Examples of Block Cipher: AES, DES, RSA

Key Size:
All the encryption algorithms can operate in different key sizes. A cipher is no stronger than its key length: if there are too few keys, an attacker can enumerate all possible keys. These different key sizes have different impacts on hardware/software and therefore an ideal sweet spot is choosing the right key length that offers enough security and provides good performance. Usually, the longer the key size the better the security but there are possible exceptions (for example, AES).

Here is a good website to get a good idea about which keysizes are to be used based on the algorithm that you use: https://www.keylength.com/en/compare/

Key Lifetimes:
There are lifetimes associated with the keys that encryption algorithms generate (lifetime as a concept can be associated with any key generation protocol or algorithm). Although certain encryption algorithms have been proven to be secure, it is considered to be good practise to refresh keys from time to time (consider lifetime of a key to be inversely proportional to sensitivity of the data encrypted).

Mode of Operation:
Block ciphers should not be used as described in their original definitions. This is inherently weak since an adversary can build up a dictionary of different plain texts and their corresponding ciphertexts and use them to decipher an encrypted message. Besides the weakness in using encryption algorithms directly, there is also an added limitation where direct use cannot work on variable length of messages (the length of a message should be a multiple of the cipher block size in length).

There are therefore a few modes of operation:
Galois/Counter Mode (GCM)
Cipher Block Chaining (CBC)
Electronic Code Book (ECB)
Cipher Feedback (CFB)
Output Feedback (OFB)
Counter (CTR)

We strongly recommend to go through the following document to understand the below operations in greater detail. This will help in futher understanding the strengths and weakenesses of different modes.

Explaining the above modes in detail is beyond the scope of this website. CTR and GCM are

GCM and CBC modes internally work quite differently; they both involve a block cipher and an exclusive-or, but they use them in different ways. It is also important to understand that GCM is considered a authenticated encryption algorithm where authentication and encryption are combined. This is unlike the other modes described above which only do encryption.

What to choose?
Please note that there are no hard and fast rules here. These are recommendations.

Recommended Encryption Algorithm: AES, RSA or 3DES

Key Size:
NSA rates 128-bit AES as good enough for SECRET traffic; 256-bit AES is good enough for TOP-SECRET traffic.
3DES, in general, provides 168bits (three 56-bits DES Keys) and this is secure.
For RSA, NIST recommends 2048-bit keys for RSA. An RSA key length of 3072 bits should be used if security is required beyond 2030.

Lifetime: Anything from 24hrs - 30 days (based on the sensitivity of the data).

Mode of Operation: CTR or GCM. Note that if someone is using RSA, they need not be worried about mode of operation but rather mode of padding. For more information, visit here.
